# Ollama setup (Windows) — free local AI for trueform

Ollama runs a small language model **on your own computer**. No account, no API key, no subscription. This is the $0 path to real humanization quality.

## 1. Install Ollama

1. Go to [https://ollama.com/download](https://ollama.com/download)
2. Download **Windows** and run the installer
3. **Restart your terminal** (or Cursor) so `ollama` is on your PATH

## 2. Pull a model

Open a terminal and run:

```powershell
ollama pull llama3.2:3b
```

This downloads a ~2GB model. It is smaller and faster than `llama3.1`, and works well for rewriting text.

Other options:

| Model | Size | Notes |
|-------|------|-------|
| `llama3.2:3b` | ~2 GB | **Recommended** — fast, good enough for drafts |
| `llama3.1` | ~4.7 GB | Higher quality, slower |
| `phi3` | ~2.3 GB | Very fast on weaker laptops |

## 3. Verify setup

From the `trueform` folder:

```powershell
cd C:\Users\fenet\Desktop\trueform
$env:PYTHONPATH="src"
python -m trueform doctor
```

You want to see:

```
[ok] Ollama CLI: ...
[ok] Ollama server: running on http://localhost:11434
[ok] Ollama models: llama3.2:3b
```

## 4. Humanize with Ollama

```powershell
python -m trueform humanize --provider ollama --model llama3.2:3b --file examples\sample.txt
```

The first run may take 30–60 seconds while the model loads. Later runs are faster.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `ollama` not recognized | Restart terminal after install; open Ollama app from Start menu |
| `Could not reach Ollama` | Open the Ollama app (it runs in the system tray) |
| `model not found` | Run `ollama pull llama3.2:3b` |
| Very slow | Use a smaller model (`llama3.2:3b`) or `--single-pass` |
| Out of disk space | Models need 2–5 GB free; delete unused models with `ollama rm <name>` |

## How it connects to trueform

```
trueform CLI  -->  OllamaProvider  -->  http://localhost:11434/api/chat  -->  local model
```

No data leaves your machine. The same multi-pass loop and scoring work exactly as with `mock`, but the rewrites are generated by a real language model.
